本篇是关于Hexo的基本操作步骤.

部署步骤

每次部署的步骤，可按以下三步来进行。

一些常用命令：

hexo new “postName” #新建文章

hexo new page”pageName” #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

hexo deploy #将.deploy目录部署到GitHub

hexo help # 查看帮助

hexo version #查看Hexo的版本

发表博文

  1. 首先cd blog, 进入blog文件夹下， 使用 hexo new “postName” #新建文章
  2. 在source/_post/ 中 会产生 postName.md文件, 直接双击打开 书写博文，保存
  3. 使用 hexo server 本地浏览效果
  4. 使用 hexo clean, hexo generate 和 hexo deploy 部署三部曲 把当前文章部署到服务器上，即完成文章发布。

更换主题

  1. 首先cd blog, 进入blog文件夹下，比如我这里用的是 maupassant 主题
  2. git clone https://github.com/icylogic/maupassant-hexo.git themes/maupassant ，直接会把主题安装到themes/ 下. 也可能需要安装其他支持文件：
  1. 然后修改blog文件夹下 _config.yml 中， theme : maupassant

具体可以参照： here maintained by @tufu9441.
