Day4 Difference between Comparator and Comparable

Aug 8, 2016 | comparator, comparable | Hits

本篇是关于Comparator和Comparable的比较和总结.

攻略模板：

  • 回忆内部结构：分析结构特性，比较两者差异
  • 学习、穷解该结构和概念
  • 总结该概念特点，并总结算法题

自己回忆：

  • Comparator: interface,可以直接在方法的后面重写和定义。
  • Comparable: interface, 在类中实现并继承。

学习过程：

  • Comparable: A comparable object is capable of comparing itself with another object, like a.compareTo(b). The class itself must implement the java.lang.Comparable interface in order to be able to compare its instances. 排序接口，若一个类实现了comparable接口，就意味着“该类支持排序”。相当于一个“内部比较器”。
  • Comparator: A comparator object is capable of comparing two different objects, like compare(a, b). The class is not comparing its instance, but some other class’s instances. The comparator class must implement the java.util.Comparatorinterface. 比较器，我们若需要控制某个类的次序，可以建立一个“该类的比较器”来进行排序。相当于一个“外部比较器”。
  • 关于排序顺序,ascending and descending
  • compare: is from the Comparator interface.
  • compareTo: is from the Comparable interface.
应用PriorityQueue
  • PriorityQueue是利用堆排序Heap sort原理，使用最大堆Max-heap或者最小堆Min-heap，默认最小堆，即每次取出的是最小值，因此每次出队和入队操作不能保证队列元素的有序性，只能保证队头元素和新插入元素的有序性。
    PriorityQueue is implemented as a Binary Heap,both inserting and removing the values through offer() and poll() methods have O(lg K) complexity.
