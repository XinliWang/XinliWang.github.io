Day6 DataStructure Collection

Aug 13, 2016 | Data Structure,Collection | Hits

本篇是关于Collection的各种数据结构的分析和对比.

攻略模板

  • 回忆数据结构：分析数据结构特性，比较类似数据结构差异
  • 实现数据结构
  • 实战训练使用该数据结构
  • 总结该数据结构特点，并总结算法题

自己回忆

  • Collections and Arrays: 都可以快速调用实现sort功能
  • Queue: First in first out.
  • Stack: First in last out.

实现数据结构

学习后总结

Collections vs. Arrays

  • Collections: Collections.sort(list) , 它是专门operates on List排序。使用的算法为TimSort(modified by MergeSort) and ensure it’s stable.
  • Arrays: Arrays.sort(array) , 它是专门为operates on Array排序。使用的算法为QucikSort for primitive type(not stable). TimSort for objects(stable). Actually, java.util.Collections.sort() simply calls java.util.Arrays.sort() to do the heavy lifting.

Queue vs. Stack

  • Queue is an interface, if you want to get this collection, you need a specific implementing class, like LinkedList or ArrayDeque.
  • Stack is a class. It implements List.
  • operation:

Stack:

  • pop() 删去element at the top(head/first) of stack and return. Throws EmptyStackException if the stack is empty.
  • push() 存入element at the top(head/first) of stack.
  • peek() 取出element at the top(head/first) of stack. Throws EmptyStackException if the stack is empty.

Queue:

  • boolean add() vs. boolean offer() : insert the element into the tail(end/last) of the queue. add() throws an IllegalStateException if no space is available.
  • poll() 删去element at the top(head/first) of stack and return, return null if queue is empty.
  • peek() 取出element at the top(head/first) of stack, return null if the queue is empty.
  • empty() 返回if queue is empty.

Deque:

  • addFirst(), offerFirst()
  • addLast(), offerLast()
  • pollFirst(), peekFirst()
  • pollLast(),peekLast()
  • removeFirst(), removeLast()

ArrayDeque vs. LinkedList

Both implement Deque and Queue.

  • Memory cost: ArrayDeque is memory efficient since you don’t have to keep track of next node unlike in LinkedList.

  • Accessing speed: Accessing an element in ArrayDeque is always faster than LinkedList. It takes O(1) for accessing elements. In Linked List, it will take O(n) to find last element.

  • ArrayDeque: This class is likely to be faster than Stack when used as a stack, and faster than LinkedList when used as a queue. It doesn’t support null elements and concurrent access by multiple threads and is not thread-safe.
